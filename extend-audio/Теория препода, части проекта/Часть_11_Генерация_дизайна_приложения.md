## **Генерация дизайна мобильного приложения**

От идеи и ТЗ — к набору экранов, которые можно использовать для схем пользовательских путей и дальнейшей разработки.

| Главная мысль Нейросеть не заменяет проектирование. Она помогает быстро получить первый визуальный вариант, который студент затем проверяет, уточняет и оформляет как рабочий материал проекта. |
|---|

| ТЗ | Промт | Дизайн | Отбор экранов | Notion / Buildin | Схемы путей |
|---|---|---|---|---|---|

| Что должно получиться | Минимальный результат |
|---|---|
| Набор экранов | 6-10 экранов мобильного приложения |
| Единая стилистика | цвета, шрифты, карточки, кнопки и навигация выглядят согласованно |
| Оформление | страница в Notion / Buildin со скриншотами, таблицами и ссылками |
| Подготовка к следующей части | экраны имеют названия и ID, чтобы по ним построить пользовательские пути |

*Важно: оцениваться будет не только «красота картинки», а логика экранов, связь с MVP и аккуратность оформления результата.*

# **Задача студента**

На основе своего MVP, требований и технического задания нужно сгенерировать дизайн мобильного приложения. Дизайн должен помогать понять, как пользователь будет работать с продуктом.

| Не просто красивая картинка Экран должен решать конкретную задачу пользователя: войти, найти, добавить, выбрать, сохранить, посмотреть результат. | Не набор случайных экранов Все экраны должны быть связаны между собой и относиться к одному приложению. |
|---|---|
| Не готовый финальный UI Первый вариант можно и нужно дорабатывать: менять подписи, блоки, цвет, порядок, навигацию. | Не копия чужого шаблона Можно использовать генерацию, но итог должен быть адаптирован под собственную идею проекта. |

# Предлагаемый** набор экранов**

Выберите экраны под свой проект. Если какой-то экран не подходит по смыслу, замените его на аналогичный.

| ID | Экран | Зачем нужен |
|---|---|---|
| SCR-01 | Стартовый экран / onboarding | коротко объясняет, что это за приложение и для кого оно |
| SCR-02 | Регистрация или вход | показывает, как пользователь попадает в систему |
| SCR-03 | Главный экран | основная точка входа после запуска приложения |
| SCR-04 | Список / каталог / лента | показывает набор объектов: задачи, курсы, записи, товары, события |
| SCR-05 | Детальная страница | показывает подробности выбранного объекта |
| SCR-06 | Создание или редактирование | форма добавления, изменения или настройки объекта |
| SCR-07 | Профиль / личный кабинет | данные пользователя, прогресс, настройки или история |
| SCR-08 | Состояние ошибки / пустой экран | что видит пользователь, если данных нет или произошла ошибка |

# **Где можно сгенерировать дизайн**

| Инструмент | Когда использовать | Комментарий |
|---|---|---|
| Stitch by Google | основной вариант | подходит для генерации UI мобильных и веб-приложений по текстовому описанию |
| Figma Make | если есть доступ в Figma | удобно, когда дальше дизайн будут дорабатывать в Figma |
| Uizard | быстрые прототипы | может помочь собрать первые экраны и вайрфреймы |
| Visily | структурные экраны | удобен для черновиков, dashboard-логики и простых экранов |
| Figma / Penpot вручную | если генерация не подошла | можно собрать дизайн вручную по своему ТЗ и референсам |

Основная ссылка: 

*Stitch лучше просить не «сделай красивое приложение», а давать ему роль продукта, аудиторию, список экранов, стиль и ограничения.*

# **Платный доступ**

| Позиция для студентов Покупка подписки не является обязательной. Задание можно выполнить бесплатными инструментами или вручную. Платный доступ может дать больше попыток, моделей и возможностей, но не должен влиять на оценку. |
|---|

По желанию студент может использовать расширенный доступ к Gemini / Google AI Pro. На сторонних площадках, например в разделе Gemini на Plati.Market, встречаются предложения примерно от 700-800 рублей за 6 месяцев, но цены, условия и риски зависят от конкретного продавца.

Раздел Plati.Market для самостоятельной проверки: 

*Перед покупкой нужно самостоятельно проверить отзывы, условия доступа, безопасность аккаунта и правила площадки. Для выполнения учебного задания покупка не требуется.*

# **Как подготовить хороший промт**

Перед генерацией соберите вводные. Чем точнее описание, тем меньше случайного результата.

| Что указать | Пример |
|---|---|
| Тип приложения | мобильное приложение для учета личных финансов |
| Целевая аудитория | студенты 16-20 лет, которым нужно быстро видеть расходы |
| Основные действия | добавить расход, посмотреть статистику, выбрать категорию |
| Экраны | onboarding, вход, главная, добавление расхода, статистика, профиль |
| Стиль | чистый интерфейс, светлая тема, спокойные цвета, без детских иллюстраций |
| Ограничения | русский текст, крупные кнопки, понятная нижняя навигация |

| Design a clean mobile app UI for [название проекта]. Target users: [целевая аудитория]. Main goal: [главная цель приложения]. Create screens: [список экранов]. Core features: [ключевые функции из MVP]. Style: [цвета, настроение, уровень строгости]. Use Russian text in the interface. Make navigation consistent and easy to understand. Avoid overloaded screens and decorative elements that do not help the user. |

Use Russian text in the interface. Make bottom navigation consistent across main screens. |

# **Порядок выполнения**

1. Откройте свое ТЗ и MVP. Выпишите главную цель приложения и функции Must Have.

2. Составьте список экранов. У каждого экрана должен быть ID: SCR-01, SCR-02 и так далее.

3. Выберите инструмент генерации. Основной вариант — Stitch by Google; можно использовать альтернативу.

4. Составьте первый промт по шаблону из этой части.

5. Сгенерируйте дизайн и сохраните результат: скриншоты, экспорт или ссылку.

6. Сделайте минимум две итерации: исправьте стиль, тексты, экранный состав или навигацию.

7. Оформите страницу в Notion / Buildin по структуре из задания.

| Совет Лучше получить простой, но логичный дизайн, чем красивый набор экранов, по которому невозможно понять путь пользователя. |
|---|
