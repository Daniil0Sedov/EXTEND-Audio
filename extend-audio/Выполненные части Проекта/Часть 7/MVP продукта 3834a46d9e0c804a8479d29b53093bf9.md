# EXTEND Audio

EXTEND Audio — кроссплатформенный аудиоплеер с сообществом пресетов. Приложение помогает пользователям быстро настроить звук под конкретную модель наушников через готовые пресеты от других пользователей или собственный эквалайзер. Создается для аудиофилов, геймеров и меломанов, которые хотят качественный звук без ручной возни с настройками.

## Определение MVP

MVP - это не готовый проект, а минимально рабочая версия продукта, в которой есть необходимый функционал для теста. Для моего проекта это означает то что: пользователь может зайти в приложение, загрузить папку со своей музыкой затем послушать ее и настроить точно звук по своему предпочтению в точном эквалайзере где он сможет сохранить пресет под какую то конкретную модель и прослушать. Всё остальное (публикация пресетов их рейтинг и профиль) это все позже.

## Целевая аудитория MVP

- Аудиофилы — люди, которые следят за качеством звука и готовы тратить время на настройку, но хотят быстрый результат.
- Геймеры — пользователи игровых наушников (JBL Quantum, Razer, HyperX, Logitech), которые хотят получить оптимальный звук для игр.
- Меломаны  и обычные люди — все, кто слушает музыку и раздражён заводским звуком «из коробки».

## Проблема пользователя

Люди зачастую покупают дорогие наушники, но заводской звук их может не устраивать. Ручная настройка эквалайзера отнимает время и требует понимания частот. В итоге человек молчит с плохим звуком или тратит много времени на эксперименты.

## Гипотеза решения

«Если мы сделаем..., то пользователь сможет...»:
Если мы сделаем аудиоплеер с поиском готовых пресетов под конкретные модели наушников и встроенным эквалайзером, то пользователь сможет за несколько нажатий найти и применить оптимальные настройки звука под свою модель, сразу прослушав на своем плейлисте.

## Главная ценность

Пользователь получает качественный звук в своих наушниках имея возможность точно настроить звук, музыка начинает звучать именно так, как он того хочет.

## Основной сценарий

1	Открывает приложение
2	Вводит модель своих наушников
3	Видит точный и понятный эквалайзер
4	Сохраняет пресет и нажимает «Применить»
5	Загружает трек из своей библиотеки
6	Слушает трек с новым пресетом
7	Если не нравится — пробует другой пресет

## Функции MVP

| № | Функция | Категория | Обоснование | Результат |
| --- | --- | --- | --- | --- |
| 1 | Создание пресета под модель наушников | Must-have | Ядро продукта — без этого MVP не проверит идею | Пользователь создаст пресет для какой то модели наушников тем самым пополнив библиотеку. |
| 2 | Применение пресета | Must-have | Без применения пресет бессмысленен | Звук меняется мгновенно |
| 3 | Встроенный эквалайзер | Must-have | Позволяет подстроить звук вручную если готовый пресет не подходит | Точная настройка частот |
| 4 | Прослушивание треков | Must-have | Без этого нельзя проверить результат | Пользователь слышит разницу |
| 5 | Загрузка треков из устройства | Must-have | Нужны свои файлы для тестирования | Работа с личной библиотекой |
| 6 | Сохранение своего пресета | Should-have | Полезно, но MVP работает и без этого | Пользователь может вернуться к настройкам |
| 7 | Библиотека треков | Should-have | Удобно, но можно обойтись загрузкой по одному | Организация музыки |
| 8 | Профиль пользователя | Could-have | Не критично для первой версии | Личный кабинет |
| 9 | Рейтинг пресетов | Won't-have now | Улучшает качество, но не обязательно для MVP | Понятно, какие пресеты лучшие |
| 10 | Push-уведомления | Won't-have now | Не нужны для проверки основной идеи | - |
| 11 | Чат / комментарии | Won't-have now | Социальная функция — вторая версия | - |
| 12 | Облачная синхронизация | Won't-have now | Данные хранятся локально | - |

## Что не входит в MVP

- Регистрация через внешние сервисы (Google, Apple)
- Публикация пресетов в общий доступ
- Комментарии к пресетам
- Рейтинг пресетов
- Облачная синхронизация между устройствами
- Полноценная аналитика прослушиваний
- Кастомизация интерфейса (темы, шрифты)

## Схема MVP

[Проблема: плохой звук "из коробки"]
↓
[Гипотеза: создание пресета под свои наушники решают проблему]
↓
[Пользователь: аудиофил / геймер / меломан]
↓
[Главный сценарий: создание → применение → проверка]
↓
[MVP: создание пресетов + эквалайзер + плеер]
↓
[Проверка: пользователь прошёл сценарий?]
↓
[Да → развитие продукта / Нет → доработка]

## Критерии готовности

| Блок проверки | Что должно быть |
| --- | --- |
| Запуск | Приложение открывается без ошибок |
| Создание | Пользователь создает пресет под модель наушников  |
| Сохранение | После нажатия сохранить пресет появляется в локальной библиотеке |
| Эквалайзер | Пользователь может двигать ползунки частот и другие настройки звука |
| Прослушивание | Пользователь включает трек и слышит его с созданным пресетом |
| Документация | Есть ТЗ, ссылка на GitHub, документация Notion |

## Критерии успеха

| Критерий | Как проверить |
| --- | --- |
| Пользователь создал пресет | Вводит модель → видит эквалайзер → сохраняет |
| Звук изменился | После применения пресета слышна разница |
| Пользователь понял сценарий | Не нужна инструкция для прохождения главного пути |
| Обратная связь | После демонстрации минимум 2–3 человека подтверждают пользу |
| Основной сценарий пройден | От открытия до прослушивания — не более 5 действий |

## Итоговый MVP

MVP продукта EXTEND Audio — это первая рабочая версия аудиоплеера, которая позволяет пользователю создать собственный пресет звука под конкретную модель наушников через встроенный эквалайзер и сразу проверить результат на своём треке. В первую версию входят: создание пресета, эквалайзер, загрузка и прослушивание треков из локальной библиотеки. Не входят: публикация пресетов в общий доступ, чат, рейтинг, облачная синхронизация и профиль — эти функции появятся после подключения к единой базе данных и создания сообщества. Такой MVP логично делать первым, потому что он позволяет проверить: нужна ли пользователям точная настройка звука и помогает ли эквалайзер решать проблему заводского звука «из коробки». После обратной связи и доработки эквалайзера будет логично перейти к интеграции с общей базой пресетов и построению сообщества.
